Git sam po sebi je odlicna stvar, ali mnogi ga potcenjuju, moze dosta da pomogne, ali ako se koristi nestrucno moze doci do
katastrofalnih posledica. Zato se treba, na pocetku, pre ucenja bilo koje tehnologije, upoznati sa osnovnim funkcijama koje
git nudi. Termini koje treba znati na pocetku:

- clone
- commit
- checkout
- push
- pull
- pull request
- merge
- rebase
- branch
- gitignore
- fork

https://www.atlassian.com/git/tutorials/comparing-workflows

Kada se sve ovo prodje trebalo bi simulirati rad na github-u ili bitbucket-u. Clanovi grupe treba da naprave projekat
od kojih ce svako menjati svoj deo koda. Treba se odrediti po kom workflow sistemu ce biti radjen projekat i tako poceti.
Svaki od ucesnika ce imati svoju granu u koju ce menati neki deo koda i ponovo push-ovati na master granu.
Simulirati u svom radnom okruzenje resavanje konflikta.
